# Fleet operations

How the multi-agent **worker fleet** is composed, spawned, drained, and
recovered. Companion to the multi-machine bus bridge in
[`README.md`](../README.md).

**Repo:** [github.com/chendren/muster-fleet](https://github.com/chendren/muster-fleet)

## Mental model

```
                    ┌──────────────────────────────────────┐
                    │         muster bus (hub only)        │
                    │   sock + bus.db  (tasks / inbox)     │
                    └───────────────┬──────────────────────┘
           ┌────────────────────────┼────────────────────────┐
           ▼                        ▼                        ▼
   headless hub Groks         hub Claude TUI           spoke Grok TUIs
   grok-hub-a/b/c             hub-tui-claude           grok-spoke-a/b
   (npm grok -p loops)        (tmux, local)            (tmux over SSH)
           │                        │                        │
   self-poll every ~10s      fleet-nudge-tui         fleet-nudge-tui
   early-kill on idle        local tmux kick         remote SSH send-keys
```

| Worker | Machine | Mode | How it picks up work |
|--------|---------|------|----------------------|
| `grok-hub-a/b/c` | hub | headless loop | `muster-loop-*.sh` runs `grok -p` every ~8–15s |
| `hub-tui-claude` | hub | tmux TUI | `fleet-nudge-tui` → `muster nudge` + `tmux send-keys` |
| `grok-spoke-a/b` | spoke | tmux TUI | `fleet-nudge-tui` → SSH `tmux send-keys` on `muster-remote` |
| `grok-mbp-worker` | spoke | optional headless | separate spoke loop (if deployed) |

**Headless hub workers have no live pane.** `muster gc` may tombstone them
as “dead session” because there is no tmux pane. That is expected. Each
drain cycle re-calls `register_agent`, so they stay useful. Do **not** treat
`muster agents` “LIVE ✗” as “worker is broken” for hub Groks.

## Install fleet scripts

From the clone (hub):

```bash
cd /path/to/muster-fleet
install -m 755 fleet/muster-spawn-tui.sh        ~/.local/bin/muster-spawn-tui
install -m 755 fleet/fleet-restart-hub-workers.sh ~/.local/bin/fleet-restart-hub-workers
install -m 755 fleet/fleet-nudge-tui.sh         ~/.local/bin/fleet-nudge-tui
```

Canonical source lives in this repo under `fleet/`. Redeploy after every
pull if you use the installed copies.

## Spawn a worker

```bash
muster-spawn-tui <claude|grok> <hub|spoke> <alias> [role]
```

Examples:

```bash
# Three hub Groks (headless drain loops) + one Claude TUI
muster-spawn-tui grok   hub   grok-hub-a worker
muster-spawn-tui grok   hub   grok-hub-b worker
muster-spawn-tui grok   hub   grok-hub-c worker
muster-spawn-tui claude hub   hub-tui-claude worker

# Spoke Groks (real TUIs on the MacBook)
muster-spawn-tui grok spoke grok-spoke-a worker
muster-spawn-tui grok spoke grok-spoke-b worker
```

| CLI | Machine | What actually starts |
|-----|---------|----------------------|
| `claude` | `hub` | tmux session named **exactly** the alias; Claude with full permission bypass + workspace trust pre-accepted |
| `grok` | `hub` | **not** a TUI — persistent headless poll loop (`/tmp/muster-loop-<alias>.sh`) |
| `grok` | `spoke` | real tmux TUI via SSH (`muster-remote`) using native/Rust grok |
| `claude` | `spoke` | **refused** — subscription keychain not available over SSH |

### Why hub Grok is headless

npm `@vibe-kit/grok-cli` interactive mode corrupts multi-tool-call argument
streams (delta merge keyed by array position instead of OpenAI `index`).
Headless `-p` is reliable. Spoke uses the native/Rust grok build, so TUI is
fine there.

### Session name = alias

tmux session name is **the alias** (not `muster-tui-hub-…`). SessionStart
hooks register under the session name when `MUSTER_ALIAS` is unset; a
mismatched name produces **two agents on one pane** and double-counts on
the dashboard.

## Drain architecture (why “only the operator works”)

Three stacked failure modes made the fleet look idle:

1. **Unquoted heredoc** in the loop writer expanded `"$@"` at write time →
   every cycle ran `"" &` → `: command not found` (rc=127). Fixed with a
   quoted `<<'LOOP_EOF'` body + env preamble.
2. **`grok -p` never exits** after the model says `idle` → workers sat until
   a 180s hard timeout → ~3 minute poll interval. Fixed with **early-kill**:
   watch the cycle log for `idle` / `FLEET_CYCLE_N_DONE` and kill the node
   process (~10–15s cycles).
3. **TUI workers only act on typed input.** Register-and-wait primes left
   Claude/spoke sitting forever. Fixed with **`fleet-nudge-tui`**.

### Headless loop details

- Script: `/tmp/muster-loop-<alias>.sh` (regenerated by `muster-spawn-tui`)
- Log: `/tmp/muster-loop-<alias>.log`
- Pid: `/tmp/muster-loop-<alias>.pid`
- Max wall clock per cycle: 120s (safety net only)
- Idle sleep between cycles: 8s
- Binary: always `$HOME/.npm-global/bin/grok` (never bare `grok` + PATH)

Each cycle:

1. `register_agent`
2. `get_inbox`
3. Claim/complete open tasks; handle unread action messages
4. Print `idle` + `FLEET_CYCLE_N_DONE` and stop (one pass, no tool thrash)

### TUI nudge supervisor

```bash
fleet-nudge-tui          # forever, every 20s (default)
fleet-nudge-tui once     # single pass
```

| Target | Mechanism |
|--------|-----------|
| `hub-tui-claude` | `muster nudge` + local `tmux send-keys` into session |
| `grok-spoke-a` | SSH `muster-remote` → `tmux send-keys` to `muster-tui-spoke-grok-spoke-a` |
| `grok-spoke-b` | SSH → session `grok-spoke-b` |

**Do not rely on `muster nudge` for spoke aliases.** Spoke agents often
store hub-side socket paths in `bus.db`; local nudge can hit the wrong pane
(even Claude’s). Remote SSH kick is the reliable path.

Logs: `/tmp/muster-fleet-nudge.log`  
Pid: `/tmp/muster-fleet-nudge.pid`

## Restart the hub fleet

```bash
fleet-restart-hub-workers
```

Kills prior hub loops by pidfile (not a self-matching `pkill -f`),
re-spawns `grok-hub-a/b/c`, restarts the nudge supervisor, and runs one
immediate nudge so open tasks don’t wait a full interval.

## Recommended composition

A proven mix on this hardware:

| Alias | CLI | Machine | Role |
|-------|-----|---------|------|
| `MacStudioGrok1` | grok (interactive operator) | hub | producer / orchestrator |
| `grok-hub-a` | grok headless | hub | worker |
| `grok-hub-b` | grok headless | hub | worker |
| `grok-hub-c` | grok headless | hub | worker |
| `hub-tui-claude` | claude TUI | hub | worker (visible pane) |
| `grok-spoke-a` | grok TUI | spoke | worker |
| `grok-spoke-b` | grok TUI | spoke | worker |

## Permissions (never spawn a bare CLI)

| CLI | Required launch flags / prep |
|-----|------------------------------|
| Claude Code | `--dangerously-skip-permissions` **and** `--permission-mode bypassPermissions`; pre-set `hasTrustDialogAccepted` in `~/.claude.json` for `$HOME` |
| Native grok | `--permission-mode bypassPermissions` |
| npm grok headless | `-p` only (no interactive permission mode) |

Skip-permissions alone can still leave Claude in **manual mode** (footer
toggle), which hangs on the first MCP call.

## Smoke test: prove the fleet drains

```bash
# From any agent with muster MCP (example: producer MacStudioGrok1)
# Create one task per worker, e.g.:
#   subject: DRAIN-PROOF-A
#   body: write /tmp/fleet-drain-a.txt ; reply ; task_transition completed

# Watch
muster tasks grok-hub-a
muster tasks hub-tui-claude
ls -la /tmp/fleet-drain-*.txt
ssh muster-remote 'ls -la /tmp/fleet-drain-*.txt'

# Logs
tail -f /tmp/muster-loop-grok-hub-a.log
tail -f /tmp/muster-fleet-nudge.log
```

Healthy headless cadence after early-kill:

```
cycle=N begin
… register + drain tools …
cycle=N end rc=0          # typically 8–20s later, not ~180s
```

## Operator checklist when “fleet isn’t working”

1. **Hub loops alive?**  
   `ps -p $(cat /tmp/muster-loop-grok-hub-a.pid)` and `grep cycle= /tmp/muster-loop-grok-hub-a.log | tail`
2. **Still getting `: command not found`?**  
   Loop script is pre-fix — redeploy `muster-spawn-tui` and `fleet-restart-hub-workers`.
3. **Cycles take ~3 minutes?**  
   Early-kill missing — redeploy loop writer; kill stuck `node …/grok -p` processes.
4. **Claude has open tasks but idle?**  
   `fleet-nudge-tui once` or check `/tmp/muster-fleet-nudge.log`.
5. **Spoke silent?**  
   `ssh muster-remote 'tmux ls'` and remote kick log lines in the nudge log.
6. **After `muster gc`, hub Groks show LIVE ✗?**  
   Expected for headless. Wait one cycle for re-register; don’t spam gc on headless fleets.
